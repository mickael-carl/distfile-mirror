Introduction

The Distfile Mirror service is a generic tool for mirroring artifacts from the Internet, so that they are available internally. The goals for having such a service are as follows:

  • To improve the speed and reliability at which these artifacts are available. It allows build processes that automatically download third-party artifacts (e.g., Bazel, Buildroot) to work efficiently.
  • To reduce the amount of Internet bandwidth consumed.
  • To provide permanent, durable storage of these artifacts. The Internet provides no guarantee that artifacts remain available, nor whether they remain unmodified. The intent of this service is to keep files stores indefinitely once they are downloaded. Depending on such resources directly risks reproducibility of work.

As this service provides no forms of authorization, it may only be used to mirror artifacts that are publicly available.

Using this service

The Distfile Mirror service consists of two user visible parts:

  • The administration page (i.e., the page you are visiting) that can be used to explore artifacts that are currently stored. It can also be used to set up new artifacts that need to be mirrored. Artifacts may be managed by type (e.g., plain files, Docker container images) by choosing one of the menu options at the top of this page.
  • A HTTP proxy that can be used to access the artifacts stored by this service through their original URLs. Any SSL connection established through this service uses an auto-generated certificate that is signed by the same Certificate Authority.

Many Linux-based applications may be configured to access the HTTP proxy by setting the following environment variables:

export HTTP_PROXY={{.ProxyPublicAddress}}
export HTTPS_PROXY={{.ProxyPublicAddress}}
